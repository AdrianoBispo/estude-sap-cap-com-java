# Módulo 02 (Parte 5): Contextos e Transações

![Infográfico sobre o RequestContext](../Infograficos/02.05.01_RequestContext_Maestria.png)

Este módulo aborda dois conceitos fundamentais para o desenvolvimento de aplicações robustas: o gerenciamento de estado da requisição através de contextos e o controle de transações para garantir a integridade dos dados.

## Conteúdo

| Documento | Infográfico | Descrição |
| :--- | :--- | :--- |
| [Maestria em RequestContext](./02.05.01_RequestContext_Maestria.pdf) | [Visualizar](../Infograficos/02.05.01_RequestContext_Maestria.png) | Domine o `RequestContext`, o objeto que carrega informações vitais da requisição (usuário, tenant, locale) e que está disponível em todo o ciclo de vida do evento. |
| [Transações com ChangeSetContexts](./02.05.02_Transacoes_ChangeSetContexts.pdf) | [Visualizar](../Infograficos/02.05.02_Transacoes_ChangeSetContexts.png) | Aprenda como o CAP gerencia transações de banco de dados automaticamente através do `ChangeSetContext`, garantindo que múltiplas operações de escrita sejam atômicas (tudo ou nada). |
