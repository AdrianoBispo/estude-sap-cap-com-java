# Módulo 1 (Parte 1): Fundamentos e Modelagem - Filosofia e Intenção

![Infografico - Modelagem de Dominio com Intenção](../Infograficos/01.01.02_Modelagem_com_Intencao.png)

Este diretório inicia nossa jornada no SAP Cloud Application Programming Model (CAP), focando nos conceitos fundamentais que guiam o desenvolvimento de aplicações robustas e eficientes. Aqui, exploramos a "filosofia" por trás do CAP, a importância da modelagem orientada pela intenção e a estrutura do Core Data Services (CDS).

## Tópicos Abordados

Os documentos a seguir apresentam os pilares conceituais do SAP CAP:

1. **A Jornada do Construtor (`01.01.01_Jornada_do_Construtor.pdf`):** Apresenta uma visão geral do ciclo de vida de desenvolvimento com CAP, desde a concepção até a implantação.

2. **Modelagem com Intenção (`01.01.02_Modelagem_com_Intencao.pdf`):** Discute como a modelagem de dados no CAP vai além da estrutura de banco de dados, capturando a semântica e a intenção do negócio.

3. **Dominando SAP CAP e CDS (`01.01.03_Dominando_SAP_CAP_CDS.pdf`):** Um mergulho profundo no Core Data Services (CDS), a linguagem que unifica a definição de modelos de dados, serviços e interfaces de usuário.

4. **A Jornada do Desenvolvedor (`01.01.04_Jornada_do_Desenvolvedor.pdf`):** Foco na experiência prática do desenvolvedor, detalhando as ferramentas e os fluxos de trabalho para criar e estender aplicações CAP.
